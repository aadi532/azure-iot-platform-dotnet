    Azure Blobs

    Information Stored

    Following lists the resources/entities/objects that are stored in Azure Blobs in Serenity

    • Device Upload files
    • Packages firmware files
    • Rules and Device Group CSV files for stream analytics job input.

    Instance Details

    The hierarchy of resources in Azure Blob is in the following order:

    • Azure Storage account
    • Containers

    The following table depicts the timeframe or moment resources are created.

    Resource Instance Level Description
    Azure Storage account Environment An Azure Storage Account is created for each environment
    Containers Tenant Containers are created for each Tenant, based on the data that is being stored.

    Naming Conventions

    As part of Serenity platform, several databases and containers are created to serve the needs of various data that is stored.

    Containers

    The following table depicts the list of containers that are created for each tenant in Serenity platform and their naming convention.

    container Description Naming Convention Example
    General Purpose Stores the uncategorised file uploaded by the system and also used for to store alerting device group CSV file {tenantid} pcs-1ebea982-68a3-483a-9c35-19015e7e8f16
    Device Upload file Stores the files uploaded by IoT Devices {tenantid}-iot-file-upload 233a1ca2-6855-43c5-8b9c-c7f85a1dd520-iot-file-upload
    Package firmware Stores the package firmware files uploaded when creating package {tenantid}-software-package 233a1ca2-6855-43c5-8b9c-c7f85a1dd520-software-package

    Access Information

    Serenity Platform APIs communicate with CosmosDB using the AccessKeys and URL of the Cosmos Account.

    CosmosDb Connection is stored in KeyVault Secrets with Key as Global--CosmosDb--DocumentDbConnectionString
