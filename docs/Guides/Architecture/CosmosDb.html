    Cosmos DB

    Azure Cosmos DB is an Fast NoSQL database with SLA-backed speed and availability, automatic and instant scalability.

    Serenity Platform uses CosmosDB as the primary data storage for entities which are not part of IoTHub.

    Information Stored

    Following lists the resources/entities/objects that are stored in CosmosDB in Serenity

    • Device Groups
    • Solution Settings
    • Device Twin Properties Cache
    • Deployments
    • Deployment Imapacted Twins
    • Deployment Imapacted Modules
    • Packages
    • Package Config Types
    • Rules
    • Alarms
    • Device LifeCycle Data
    • Device Twin Change Data
    • Device Telemetry

    Instance Details

    Hierarchy of resources in Azure Cosmos DB is in the following order:

    • Azure Cosmos account
    • Databases
    • Containers
    • Items

    Following table depicts the timeframe or moment resources are created.

    Resource Instance Level Description
    Azure Cosmos account Environment An Cosmos Account is created for each environment
    Databases Environment Databases are created at Environment level, they are specific to account.
    Containers Tenant Containers are created for each Tenant, based on the data that is being stored.
    Items Container Items are created when data is inserted into Containers.

    Naming Conventions

    As part of Serenity platform, several databases and containers are created to serve the needs of various data that is stored.

    Databases

    Following table depicts the list of databases that are created in Serenity platform and their naming convention. |Database|Description| |:--|:--|:--| |pcs-storage|Stores the Key Value based information used for Serenity UI and managed by Users.| |iot|Stores the Data related to Device telemetry, Device Lifecyle, Device Twin change and Alarms based on Telemetry.|

    Containers

    Following table depicts the list of containers that are created for each tenant in Serenity platform and their naming convention.

    container Description Naming Convention Example
    pcs Stores the Key Value based information used for Serenity UI and managed by Users for each Tenant. pcs-{tenantid} pcs-1ebea982-68a3-483a-9c35-19015e7e8f16
    alarms Stores the Alarms created by Stream Analytics Job using Telemtry and Rules alarms-{tenantid} alarms-1ebea982-68a3-483a-9c35-19015e7e8f16
    telemetry Stores the Telemetry Data from Devices processed by Message Functions telemetry-{tenantid} telemetry-1ebea982-68a3-483a-9c35-19015e7e8f16
    lifecycle Stores the LifeCycle events data of Devices processed by DeviceLifeCycle lifecycle-{tenantid} lifecycle-1ebea982-68a3-483a-9c35-19015e7e8f16
    twin-change Stores the latest Twin change event data of a Device processed by DeviceTwin function twin-change-{tenantid} twin-change-1ebea982-68a3-483a-9c35-19015e7e8f16

    Access Information

    Serenity Platform APIs communicate with CosmosDB using the AccessKeys and URL of the Cosmos Account.

    CosmosDb Connection is stored in KeyVault Secrets with Key as Global--CosmosDb--DocumentDbConnectionString

    Cost Details

    Configuration details of the CosmosDB accounts in Serenity Platform are as follows.

    Setting Value
    API SQL
    Consistency Level Strong
    Multi Region Writes False
    Automatic FailOver False

    Configuration Details for a Container are as follows |Setting|Value| |:--|:--| |Provisioned RU/s|400|

    Monthly projected cost with the above configuration will be $23.36 per container with additional charges for storage at $0.250 PerGB/Month

    Scaling

    Current configuration for Scaling is Manual. If the provisioned RU/s for each a container are not sufficient to cater the requests, Users have to manually increase the RU/s.
